# Dummy 데이터 생성 Tool PoC — CLAUDE.md

이 디렉토리는 **테스트용 Dummy Data를 생성해 저장소(DB/파일)에 추가하는 도구** PoC다.
독립된 Git Repository로 제출한다 (`DummyDataGenerator-영문이름-사번`).
전체 배경/도메인 모델은 상위 `../CLAUDE.md`를 참고.

## 목표

- `Json/` PoC에서 정한 저장 포맷(파일 경로/스키마)에 맞춰 유효한 시료(Sample)와
  주문(Order) 더미 데이터를 생성하고, 실제로 그 파일에 반영(추가)한다.
- 생성되는 데이터는 도메인 제약을 지켜야 한다.
  - 수율(Yield)은 0.0 초과 1.0 이하 범위.
  - 평균 생산시간(AvgProductionTime)은 양수.
  - 주문의 SampleId는 실제 존재하는(또는 함께 생성한) 시료를 참조.
  - 주문 상태는 `RESERVED/REJECTED/PRODUCING/CONFIRMED/RELEASE` 중 하나이며,
    상태별로 그럴듯한 조합을 만든다 (예: RELEASE 상태 주문은 CreatedAt이 과거).
- 실행 방식은 콘솔 CLI (예: 생성할 시료 수, 주문 수를 입력받거나 인자로 받음).
- 매 실행마다 기존 데이터를 덮어쓰지 않고 **추가(append)** 하는 것이 기본이며,
  초기화(clear 후 생성) 옵션은 있어도 되지만 기본 동작은 아니어야 한다.

## 설계 방향

- 난수 생성은 테스트 가능하도록 시드(seed)를 주입할 수 있게 하거나, 생성 로직
  자체(랜덤 값 조합 → 유효 엔티티 변환)를 난수 소스와 분리해 순수 함수로 둔다.
- ID 중복을 피하기 위한 ID 채번 전략을 명확히 정한다 (예: 기존 최대 ID + 1,
  또는 UUID류). 기존 데이터와 충돌하지 않는지 테스트로 검증한다.
- `Json/`의 Repository 인터페이스와 동일한 방식으로 파일에 반영한다.

## 기술 스택

- C++20, Visual Studio(.vcxproj), x64/x86
- GoogleTest는 NuGet으로 설치
- 데이터 파일 포맷은 `Json/`과 동일한 JSON

## TDD 진행 방식

1. "난수 시드 → 유효한 Sample 생성" 함수를 순수 함수로 만들고, 생성된 값이
   도메인 제약(수율 범위, 생산시간 양수 등)을 항상 만족하는지 gtest로 검증한다
   (여러 시드에 대해 반복 검증 — property-based 스타일).
2. "난수 시드 + 기존 Sample 목록 → 유효한 Order 생성" 함수도 동일하게 검증한다
   (SampleId가 항상 기존 목록에서 선택되는지 등).
3. ID 채번 로직을 기존 데이터가 있는 경우/없는 경우로 나누어 테스트한다.
4. 파일에 실제로 추가되는지(기존 데이터 보존 + 새 데이터 추가) 통합 테스트로
   검증한다.

## 범위 밖 (다른 PoC의 책임)

- 데이터 조회/모니터링은 `Monitor/`, 영속성 계층 자체의 CRUD 검증은 `Json/`의 책임.
- 이 PoC에서 검증한 생성 로직을 `Main/`의 초기 데이터 세팅 또는 테스트 지원
  도구로 통합한다.
